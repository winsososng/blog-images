## 🎯 확정된 핵심 아이디어

### 🥇 1순위: 빗 원리 — 직선을 갖다 대면 복잡한 곡선이 단순해진다
- **유형**: F (기본형 환원형) + C (환원/변환형)
- **한 문장**: "미용실에서 빗을 대고 가위질하듯, 3차함수에 1차함수(직선)를 갖다 대면 복잡한 곡선이 거리곱 형태로 단순해진다"
- **설명**: 3차함수 f(x)에서 적절한 직선 g(x)를 설정하면 f(x) = (x-a)(x-b)(x-c) + g(x) 꼴로 변환 가능. 이것은 항등식 변형일 뿐이므로 어떤 조건 계산에도 자유롭게 사용 가능. 빗 아래 부분(직선)은 무시하고, 빗 위로 튀어나온 부분(차함수)에만 집중하면 되는 구조. 이것이 "차함수 스킬"의 정의 그 자체이자 계산량 80% 감소의 원천.

### 🥈 2순위: 거리곱과의 찰떡궁합 — 차함수 스킬의 진짜 파워
- **유형**: B (관계 발견형) + C (환원/변환형)
- **한 문장**: "차함수를 빼면 거리곱 형태가 되고, 거리곱이 되면 함수값·미분계수 계산이 암산 수준으로 쉬워진다"
- **설명**: f(x) = (x-a)(x-b)(x-c) + g(x) 에서 x=d 대입 → (d-a)(d-b)(d-c) + g(d) = 거리곱 + 보정. 미분계수도 마찬가지: 차함수의 미분계수(거리곱 미분) + 직선의 기울기 = 보정. "예쁜 것(기본형)에서 관찰하고 되돌아온다"가 계산 구조의 본질. 유튜브 예시: f(6) = 5·2·1 + g(6), f'(5) = 4·1 + 3.

### 🥉 3순위: 차함수 = 관계의 서술 도구
- **유형**: E (본질 추출형)
- **한 문장**: "차함수의 본질은 두 함수 사이의 '관계'를 드러내는 것이다 — f(x) = (관계) + (대상)"
- **설명**: f(x) - g(x)를 구하면 교점 정보(관계)가 인수로 드러남. 차함수를 빼서(내려와서) 관찰하고 → 다시 되돌아가기(보정) = 기본 프로토콜. "내려오기 + 되돌아가기" 구조가 함수값 계산, 미분계수 계산 모두에 적용. 차함수 스킬이 단순 테크닉이 아니라 함수 분석의 근본 사고법임을 보여줌.

### 📌 보조 1: 왜 하필 직선(1차)인가 — 익숙함의 힘
- **유형**: A (관점 전환형)
- **한 문장**: "0차, 1차, 2차 어떤 차함수든 가능하지만, 직선을 쓰는 이유는 직선이 가장 익숙하기 때문이다"
- **설명**: "낯선 것 → 익숙하게"의 직접 적용. 빗이 곧은(=직선) 것이 포인트. 구부러진 빗으로 머리를 자르면 오히려 복잡해짐. 실전에서는 보통 1차함수이므로 앞으로 "차함수 스킬 = 1차 차함수 스킬"로 통칭.

### 📌 보조 2: 3차는 무조건 되고, 4차는 안 될 수도 있다
- **유형**: B (관계 발견형)
- **한 문장**: "3차함수는 1차항 계수를 조절하면 반드시 직선과 3번 만나게 할 수 있지만, 4차는 보장되지 않는다"
- **설명**: 3차의 도함수 = 2차(포물선) → 상수함수 y=k를 위아래로 움직이면 항상 실근 2개 구간 존재 → 원함수의 1차항 계수 조절로 극대·극소 생성 가능. 4차는 도함수가 3차 → 직선과 반드시 3번 만나는 것이 보장되지 않는 구조적 한계. 차함수 스킬의 파워와 한계를 동시에 보여줌.

## 🔑 핵심 용어 정의 (이번 포스팅부터 공식 사용)

| 용어 | 정의 |
|:---|:---|
| **차함수 스킬** | 주어진 다항함수에 적절한 1차함수(직선)를 설정하여 차함수를 거리곱 형태로 만든 뒤, 함수값·미분계수 등을 간편하게 계산하는 기법 |
| **내려오기** | f(x) - g(x)를 통해 차함수로 변환하는 과정 (빗을 갖다 대는 것) |
| **되돌아가기 (보정)** | 차함수에서 관찰한 값에 g(x) 정보를 다시 더해주는 과정 |
| **기본형** | 모든 인수가 드러나 거리곱·대칭성 등을 바로 쓸 수 있는 형태 |
